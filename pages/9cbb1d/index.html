host

rfc规范： https://www.rfc-editor.org/rfc/rfc9110.html

1、Host请求头是在http1.1作为必选被引入，如果请求头没有Host或有多个Host请求头， 将会返回400错误。 2、请求中的“Host”提供了目标URI的主机和端口信息。 3、设计Host请求头的动机： 在请求(为多个网站服务的)共享主机时，使初始服务器能够区分目标资源

简单来说： 请求头中的 host 字段是让负载节点可以区分具体请求哪个主机的，反向代理时需要注意处理
